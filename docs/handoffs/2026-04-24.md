## Non-obvious decisions worth preserving

1. **File name `process-haiku.ts` kept.** Workflow display name changed ("Process Items with Nano"), but the SDK slug `workflow('process-haiku', ...)` and the n8n workflow ID `OG4iOnuMwxoJDbvK` are unchanged. Renaming the file would have risked re-mapping or duplicate-workflow churn via the SDK→n8n deploy path. Cosmetic mismatch only; cheap to fix later if it bugs anyone.
2. **`max_completion_tokens`, not `max_tokens`.** GPT-5-family (reasoning) models reject the old `max_tokens` key. Already noted as an invariant in `HANDOFF.md` → n8n gotchas.
3. **Dropped `cache_control: { type: "ephemeral" }`.** That was Anthropic-specific. OpenAI caches long prompt prefixes automatically with no directive.
4. **System prompt folded into `messages`.** Anthropic's `system: [{ type: "text", text: ... }]` → OpenAI's `{ role: "system", content: ... }` as the first message. Text preserved verbatim.
5. **No Railway env change needed.** Studio apps don't call Haiku today (all Sonnet). `OPENAI_API_KEY` is only required in the n8n instance, and the user had already added an `OpenAI API Key` credential there. If studio tools ever migrate, add `OPENAI_API_KEY` to the Railway service env at that time — flagged in the updated Credentials section of `HANDOFF.md`.
6. **Deploy sequence**: merge PR → `update_workflow` both (saves draft) → **stop** and wait for user to bind credential in n8n UI (SDK doesn't auto-bind HTTP creds — the standing invariant) → `publish_workflow` both. Publishing before bind would 401 the first cron.
