## Decisions worth remembering

- **Studio's OAuth is self-hosted.** Studio IS the authorization server — not delegated to an external IdP. Password-gated `/authorize` page, opaque random tokens stored in Postgres (`studio_oauth_{clients,codes,tokens}`), refresh-token rotation. Single-user system so no user table. OAuth routes only mount when both `STUDIO_ADMIN_PASSWORD` and `STUDIO_BASE_URL` are set.
- **Google OAuth (for Tasks) is separate** — studio is the OAuth _client_ there. Refresh token captured via `scripts/google-auth.mjs` one-time dance, stored as `STUDIO_GOOGLE_REFRESH_TOKEN`. User confirmed their existing verified OAuth app covers this, so refresh tokens are long-lived (no 7-day test-user cap).
- **Perplexity summaries live in `news_items` with `source_type='perplexity'`.** The spec's reference to a separate `perplexity_searches` table was loose; resolved by filtering `news_items` on `source_type`. Search / momentum / thesis tools all treat Perplexity as a row variant of news.
- **Claude Desktop `"type": "http"` config fails silently** for bearer-authed remote MCPs on Mac. Two working paths: (a) OAuth via Settings → Connectors (canonical, scoped per-conversation), (b) `mcp-remote` stdio proxy in `claude_desktop_config.json` (escape hatch, global).
- **Squash-merging a stacked PR orphans the child branch.** Hit this on PRs #43/#44 this session — the child went `dirty` after the parent squash-merged because `update_pull_request_branch` couldn't auto-resolve. Recovery pattern: branch a v2 off merged main, repush the child diff. Either merge both in one PR or be ready for the re-land step. Logged in top-level HANDOFF.md "Don't do this again".
- **Scoring helpers centralized** in `apps/socialisn-studio/src/lib/scoring.js` (audience-fit regex + saturation penalty + freshness dead-windows + LIKE escape). Prevents drift between `list_daily_candidates` and `check_parking_lot` now, and between `generate_script` and future tools later.
