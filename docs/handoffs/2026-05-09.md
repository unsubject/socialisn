## Decisions worth remembering

- **Hostcountry theme picks** are a content choice. I dropped the
  econ-policy-flavoured themes (`ECON_HOUSING_PRICES`,
  `ECON_INFLATION`, `ECON_INTEREST_RATES`, `TAX_FNCACT_TAXPAYER`,
  `EPU_POLICY`) once `ECON_COST_OF_LIVING` was in — they are mostly
  redundant for diaspora middle-class concerns. Kept `IMMIGRATION`,
  `JOBS`, `ECON_COST_OF_LIVING`, `CRIME_VIOLENCE`, `EDUCATION`.
- **Rights lane** uses `sourcelang:english` instead of
  `sourcecountry:`. The original design intent was diaspora-region
  press, but GDELT's `sourcecountry:` is per-outlet not per-article
  and AND-combined with niche keywords it strips to zero. English-
  language press globally (incl. SCMP, FT, Reuters) is a better
  approximation of "transnational HK rights coverage."
- **Bench results** that informed the choice:
  - hostcountry 5×5 = 10,031 articles/24h
  - rights keywords-only = 112 articles/24h (vs 0 with the old filter)

## What I'd flag for next-session-self

- **GDELT silently fails on long queries.** ~335 chars is past the
  edge. If you ever add a lane, count the chars and stay under ~250
  to be safe. The error mode is `0 results` — no error response, just
  silent zero. Bench any new query against the live API.
- **GDELT rate-limits aggressively** against curling-from-laptop
  (1 req per 15-30s in practice, despite saying 5s). Bench on n8n's
  Hostinger IP if needed — that's where the production calls happen.
- **n8n SDK validator now stricter** than what got merged in earlier
  PRs. It rejects: `function foo() {...}` declarations, arrow
  functions (`const f = () => ...`), and post-fix string methods on
  arrays (`[...].join('\n')`). The two source files updated this
  session migrate to inline node defs + template-literal `jsCode`.
- **2 RSS sources still silently dead** (no longer crash the
  workflow but contribute nothing): `Google Trends 香港熱搜`
  (`trends.google.com/trending/rss?geo=HK`) and `Newsletter Digest`
  (`unsubject.github.io/newsletter-digest/feed`). Both probably feed-
  format issues. Not blocking. Investigate when you next need
  trending or newsletter digest signal in the briefings.
- **gdelt_signal 7-day rolling baseline is empty for ~7 days**
  starting 2026-05-09. Briefing's `articles_7d_avg` will show `'—'`.
  This is intentional (cleaner than averaging-in zeros).
